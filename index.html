Cisco CCNA · Module 6

EtherChannel

Aggregate multiple physical links into a single logical connection — gain bandwidth, redundancy, and seamless STP compatibility without wasted ports.

Link Aggregation Technology PAgP & LACP Protocols IEEE 802.3ad
The Problem

STP Blocks Redundant Links

Spanning Tree Protocol prevents Layer 2 loops by blocking redundant paths — leaving bandwidth unused and limiting throughput between switches.

  • ⚠Redundant links are blocked — wasted physical capacity
  • ⚠Single active link creates a bandwidth bottleneck
  • ⚠Slow STP reconvergence on link failure (up to 50 s)
  • ⚠No load distribution across multiple physical paths
The Solution

EtherChannel — One Logical Link, Full Power

⚡

Increased Bandwidth

Bundle up to 8 physical ports into one logical link — aggregate up to 800 Mbps (Fast) or 8 Gbps (Gigabit) of throughput.

🔄

Load Balancing

Traffic distributed across all active member links using configurable hashing methods (MAC, IP, port-based).

🛡

STP Transparency

STP sees the entire EtherChannel bundle as a single port — no redundant link is blocked, all capacity is utilized.

🚀

Fast Convergence

If one member link fails, traffic is instantly redistributed to remaining links — sub-second failover with no STP reconvergence.

Aggregation Protocols

PAgP vs LACP

PAgP

PAgP

Port Aggregation Protocol
Cisco Proprietary Layer 2
  • Developed by Cisco — works only between Cisco devices
  • Automatically negotiates EtherChannel formation
  • Supports up to 8 bundled ports per channel
  • Sends PAgP packets every 30 seconds
ModeBehavior
OnForces channel — no negotiation
DesirableActively initiates PAgP
AutoPassively waits for PAgP
LACP

LACP

Link Aggregation Control Protocol
IEEE 802.3ad Open Standard Multi-vendor
  • Open IEEE standard — works across different vendors
  • Up to 16 ports configured; 8 active + 8 standby
  • Standby ports activate automatically on failure
  • Preferred protocol for modern multi-vendor networks
ModeBehavior
OnForces channel — no negotiation
ActiveActively initiates LACP
PassivePassively waits for LACP
Best Practices

Port Requirements & Configuration Rules

01

Identical Speed & Duplex

All member interfaces must operate at the same speed and duplex setting. Mismatch prevents EtherChannel formation.

02

Matching VLAN Configuration

All ports must belong to the same VLAN (access) or be configured as trunks with identical native VLANs.

03

Consistent Allowed VLAN List

On trunk EtherChannels, the allowed VLAN list must be identical on both ends — even in auto/desirable mode.

04

Configure via Port-Channel Interface

Apply trunk, VLAN, and other settings to the logical Port-Channel interface, not individual physical ports.

05

Compatible Negotiation Modes

Modes must be compatible: Active↔Active or Active↔Passive (LACP); Desirable↔Desirable or Desirable↔Auto (PAgP). Never Passive↔Passive.

Create VLANs & Management Interface

Create required VLANs and assign management IP to VLAN 10.

S1(config)# vlan 10
S1(config-vlan)# name Management

S1(config-if)# ip address 192.168.10.11 255.255.255.0
Step 2 · Trunk Ports

Configure 802.1Q Trunks on F0/1–F0/2

Force trunking and restrict VLANs before EtherChannel formation.

S1(config)# interface range f0/1-2
S1(config-if-range)# switchport mode trunk
S1(config-if-range)# switchport trunk native vlan 1000
S1(config-if-range)# switchport trunk allowed vlan 10,20,1000
Step 3 · EtherChannel

Create LACP Port-Channel Group 1

Assign F0/1–F0/2 to channel-group 1 with LACP active mode.

S1(config)# interface range f0/1-2
S1(config-if-range)# channel-group 1 mode active

What is Po1?

Po1 (Port-Channel 1) is the virtual logical interface representing bundled ports F0/1 and F0/2. STP, VLANs, and routing interact with Po1 as a single physical link — all capacity utilized.

Troubleshooting

Common Issues & Key Takeaways

⚡

VLAN Mismatch

Ports in different VLANs or trunks with different native VLANs cannot bundle — configurations must be identical on all members.

🔀

Partial Trunk Config

Trunking set on only some member ports. Always configure trunking on the Port-Channel interface, not individual physical ports.

📋

Allowed VLAN List Mismatch

If the allowed VLAN range differs between ends, EtherChannel won't form — even in auto or desirable mode.

🤝

Incompatible Modes

Passive↔Passive or Auto↔Auto will never negotiate. At least one side must actively initiate (Active / Desirable).

⚙

Speed / Duplex Mismatch

All member interfaces must run at the same speed and duplex. Any difference prevents EtherChannel formation.

Module Summary

1
EtherChannel Aggregates Links

Bundles 2–8 ports into one logical Port-Channel. STP sees it as a single link — no ports blocked, all capacity used.

2
Two Protocols Available

PAgP (Cisco only) and LACP (IEEE 802.3ad open standard). LACP preferred for multi-vendor environments.

3
Configuration Consistency is Key

Speed, duplex, VLAN ID, native VLAN, and allowed VLANs must match on all member ports on both switches.

4
Configure via Port-Channel

Apply trunk and VLAN settings to the logical Port-Channel interface — changes automatically propagate to members.

5
Verify with show Commands

Use show etherchannel summary and show interfaces trunk to confirm correct operation.
